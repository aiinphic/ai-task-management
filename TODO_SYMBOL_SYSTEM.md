# 符號系統開發待辦清單

## 使用者需求摘要
1. ✅ 所有任務列表按優先順序排序:一級營收 → 二級流量 → 三級行政
2. 🔄 設計交通標誌風格的符號系統(純圖形、無文字)
3. 🔄 建立符號字典檔
4. 🔄 AI 根據關鍵字自動匹配符號(使用者不可修改)
5. ⏸️ 符號對照表(待符號系統確定後再製作)

## Phase 1: 研究交通標誌設計規範並設計符號系統
- [ ] 搜尋交通標誌設計規範與原則
- [ ] 分析交通標誌的視覺語言(形狀、顏色、圖案)
- [ ] 設計一級|營收符號系統(10-15個)
- [ ] 設計二級|流量符號系統(10-15個)
- [ ] 設計三級|行政符號系統(10-15個)
- [ ] 記錄符號設計規範文件

## Phase 2: 建立符號字典檔與 AI 判別規則
- [ ] 建立符號字典檔 JSON 格式
- [ ] 定義每個符號的關鍵字匹配規則
- [ ] 建立 AI 判別邏輯(關鍵字 → 符號)
- [ ] 測試判別準確度

## Phase 3: 生成符號圖形素材
- [ ] 使用 AI 生成交通標誌風格的符號圖形
- [ ] 確保符號清晰、易辨識
- [ ] 統一尺寸和風格
- [ ] 儲存為 SVG/PNG 格式

## Phase 4: 整合符號系統到任務卡片並修正排序
- [ ] 修正任務排序邏輯(按優先級排序)
- [ ] 更新任務卡片顯示符號
- [ ] 整合 AI 判別到新增任務流程
- [ ] 移除文字標題,只保留符號

## Phase 5: 測試與優化
- [ ] 測試符號辨識度
- [ ] 測試 AI 判別準確度
- [ ] 測試任務排序功能
- [ ] 優化視覺效果

## Phase 6: 交付成果
- [ ] 建立測試報告
- [ ] 儲存檢查點
- [ ] 交付使用者
