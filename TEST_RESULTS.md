## 設計特色

### 符合需求文件的核心原則

1. **時間資源量化**
   - 每個任務明確標示時間量體(10分鐘/1小時/1天)
   - 統計卡片顯示今日投入時間
   - 進度追蹤顯示日/週/月時間軸

2. **三級重要性分類**
   - 一級(營收): 橙黃漸層,視覺最突出
   - 二級(流量): 藍青漸層,中等優先
   - 三級(行政): 灰藍漸層,基礎優先
   - 每個卡片頂部清楚標示分級

3. **極簡符號化呈現**
   - 使用圖示代表任務類型(TrendingUp、Users、Settings)
   - 時間標記使用 Clock 圖示
   - 進度使用環形圖表
   - 成就使用徽章圖示

4. **遊戲化進度追蹤**
   - 環形進度圖(今日/本週/本月)
   - 成就徽章系統(連續7天、完成50任務、準時達標)
   - 完成任務時顯示鼓勵訊息
   - 清楚顯示「完成 X / Y 項任務即可下班」

5. **減少認知負擔**
   - 卡片式設計,資訊分塊清晰
   - 統一的 30-40 字任務描述
   - 顏色編碼快速識別優先級
   - 開始/結束按鈕操作簡單直觀

6. **任務執行追蹤**
   - 開始/結束按鈕記錄時間
   - 顯示開始和結束時間戳
   - 進度條視覺化任務進度
   - 完成後卡片自動消失(符合單次操作原則)

## 技術實現

### 前端技術棧
- React 19
- TypeScript
- Tailwind CSS 4 (OKLCH 色彩空間)
- Wouter (路由)
- Lucide React (圖示)
- Shadcn/ui (UI 元件庫)

### 元件架構
- `TaskCard`: 任務卡片元件(支援三種優先級、時間量體、狀態管理)
- `StatsCard`: 統計卡片元件(顯示數據和趨勢)
- `ProgressTracker`: 進度追蹤元件(環形圖 + 成就徽章)
- `Home`: 主頁面(整合儀表板和今日工作視圖)

### 設計系統
- 主色: OKLCH(0.6 0.2 270) - 深藍紫
- 輔助色: OKLCH(0.7 0.18 300) - 紫紅
- 優先級高: 橙黃漸層
- 優先級中: 藍青漸層
- 優先級低: 灰藍漸層
- 圓角: 0.75rem
- 字體: Inter
